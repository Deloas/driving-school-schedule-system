# 项目最终答辩总结

## 项目一句话介绍

驾校学员练车预约与调度管理系统——面向驾校内部练车资源协调，覆盖从预约到训练记录完整闭环的智能调度平台。

## 项目核心价值

1. **解决真实痛点**：替代电话和微信群预约方式，让练车安排有序高效
2. **聚焦核心问题**：围绕练车预约调度，不做考试、不做交管
3. **业务闭环完整**：预约→排班→调剂→完成→记录，形成完整业务链路
4. **技术方案可靠**：Redis 锁 + 条件 UPDATE 双重保障并发安全
5. **产品体验到位**：三端独立工作台，品牌统一，统计驾驶舱可视化

## 业务闭环总结

```
学员预约练车
  → 系统校验排班容量（Redis 锁 + 条件 UPDATE）
  → 主教练有空位 → 预约成功
  → 主教练满员 → 推荐调剂教练 → 调剂预约
  → 教练确认到场 → 完成练车（生成 training_record）
  → 学员已完成次数 +1
  → 学员查看训练档案
  → 管理员查看统计看板
```

## 技术亮点总结

| 亮点 | 说明 |
|------|------|
| 分布式锁防超额 | Redisson 锁 + 数据库条件 UPDATE 双重保障 |
| 主教练负责制 | main_coach_id 与 actual_coach_id 分离设计 |
| 满员调剂 | 自动推荐空闲教练，临时调剂不改变归属 |
| 练车闭环 | 完成→生成记录→累加次数，全事务保护 |
| 权限分级 | 功能模块包含权限分级，三端独立工作台 |
| 统计驾驶舱 | 教练负载、预约趋势、容量利用率可视化 |

### 3 分钟总结版

"(上接 1 分钟版)

在技术实现上，我们采用了目前业界主流的前后端分离架构。后端 Spring Boot 3.2 + MyBatis-Plus 处理业务逻辑和数据持久化，Redis + Redisson 保障并发场景下的数据一致性。前端 Vue 3 + TypeScript + Tailwind CSS 构建了三端独立工作台——管理员驾驶舱、教练工作台、学员练车工作台。

系统的一个关键设计是预约表的主教练和实际教练分离。main_coach_id 记录学员的固定归属，actual_coach_id 记录本次实际带练的教练。这个设计既支撑了主教练负责制，又通过 is_adjusted 字段支持了临时调剂机制。

在并发安全方面，我们采用的是双重保障策略：Redisson 分布式锁确保同一排班在同一时刻只有一个线程能执行预约逻辑；数据库条件 UPDATE 语句的 WHERE 条件在数据库层面阻止超额更新。即使 Redis 出现问题，数据库本身也不会允许超额。

整个项目历时约两周，从需求分析、数据库设计到前后端开发、产品优化，再到答辩文档整理，完成了 M1 到 M12 共 12 个阶段。

虽然系统还有一些可以继续完善的地方——比如消息通知、数据报表导出、更智能的调度推荐——但作为课程项目，它已经完整地展示了从需求分析到产品交付的完整软件工程流程。

以上就是我们的全部汇报。感谢老师！"
