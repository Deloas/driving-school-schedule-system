# 项目最终说明书

## 项目背景

传统驾校练车预约依赖电话沟通、微信群通知和接龙，存在以下问题：
1. 学员需频繁联系教练，预约过程不透明
2. 教练人工统计学员意向，效率低
3. 某天大量学员集中到场，每人练车时间不足
4. 另一时间只有少数学员，教练和车辆资源闲置
5. 学员、教练、车辆、时段之间缺少信息化调度规则

## 老师需求对应

| 老师要求 | 系统实现 |
|---------|---------|
| 不做考试管理 | 全程无考试预约/成绩/交管对接 |
| 学员在线预约 | 学员按日期+时段预约主教练 |
| 教练容量限制 | 每教练每半天 max_students，防超额 |
| 主教练负责制 | 学员归属主教练，预约优先主教练 |
| 满员调剂 | 主教练满员→推荐可用教练→临时调剂 |
| 车辆管理 | 车辆绑定教练、状态管理、排班占用 |
| 预约到练车完成 | 预约→到场→完成→训练记录闭环 |

## 项目目标

建设一个可演示、可扩展、具备业务规则深度的驾校练车预约与调度平台。

核心目标：
1. 替代电话/微信群预约
2. 在线按日期和时段预约练车
3. 每日容量限制防止超约
4. 主教练负责制 + 临时调剂
5. 车辆资源和占用校验
6. 练车记录沉淀和次数累计

## 核心业务流程

```
学员登录 → 选择日期和时段
  → 系统优先查主教练排班和剩余名额
  → 有名额 → 预约主教练
  → 已满员 → 推荐其他空闲教练 → 学员确认调剂
  → 生成预约记录
  → 教练查看名单 → 学员到场 → 完成练车
  → 生成训练记录 → 学员次数+1
```

## 角色权限

| 角色 | 核心能力 |
|------|---------|
| 管理员 | 教练/学员/车辆/排班/账号管理、全部预约/记录/统计 |
| 教练 | 查看预约、完成练车、标记缺席、我的学员 |
| 学员 | 预约练车、取消预约、查看训练记录 |

> 架构原则：功能模块包含权限分级，不按角色拆分重复模块。

## 功能模块

| 模块 | 核心功能 |
|------|---------|
| 教练管理 | CRUD + 状态 + 带教容量 |
| 学员管理 | CRUD + 主教练绑定 + 练车进度 |
| 车辆管理 | CRUD + 教练绑定 + 状态 |
| 排班管理 | 新增/编辑/批量生成 + 容量 + 时间段 |
| 预约管理 | 主教练预约 + 调剂推荐/预约 + 取消 |
| 并发安全 | Redisson 锁 + 条件 UPDATE |
| 练车完成 | 教练确认→生成训练记录→次数+1 |
| 统计看板 | 驾驶舱 + 教练负载 + 预约趋势 |

## 系统边界

本系统**不涉及**：
- 考试预约（科目一/二/三/四）
- 考试成绩查询
- 交管系统对接
- 缴费与合同管理
- 官方驾考全流程

## 项目价值

1. 聚焦练车预约调度，严格契合老师要求
2. 完整的预约→练车→记录业务闭环
3. Redis 分布式锁 + 条件 UPDATE 保障并发安全
4. 主教练负责制 + 调剂机制体现业务深度
5. 产品级三端前端体验
6. 适合答辩展示的清晰业务主线
