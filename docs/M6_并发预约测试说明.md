# M6 并发预约测试说明

## 测试目标

验证 Redis 分布式锁 + 数据库条件 UPDATE 双重保护下，同一排班名额被多人同时抢时不会超额。

## 前置条件

1. Docker Redis 运行中（`docker-compose up -d`）
2. 后端启动在 28081
3. 至少有一个排班 `maxStudents = 1, currentStudents = 0`

## 准备测试数据

```sql
-- 查询明天上午的排班，找一个名额充足的
SELECT s.id, s.coach_id, s.schedule_date, s.time_slot, s.current_students, s.max_students
FROM coach_schedule s
WHERE s.schedule_date = CURDATE() + INTERVAL 1 DAY
  AND s.time_slot = 'MORNING'
  AND s.status = 'OPEN'
ORDER BY s.id;
```

```sql
-- 将某个排班容量设为 1，方便测试
UPDATE coach_schedule SET max_students = 1, current_students = 0 WHERE id = <选中的排班ID>;
```

## 测试方法

### 方法 A：使用 curl 并发（Windows PowerShell）

```powershell
# 1. 先获取两个学员的 token
$body1 = @{username='student001';password='123456'} | ConvertTo-Json
$resp1 = Invoke-RestMethod -Uri http://localhost:28081/api/auth/login -Method POST -Body $body1 -ContentType 'application/json'
$token1 = $resp1.data.token

$body2 = @{username='student002';password='123456'} | ConvertTo-Json  
# 如果没有 student002，先通过 DataInitializer 或手动创建一个
```

### 方法 B：使用浏览器 Network 手动测试

1. 用 student001 登录，打开预约页
2. 用另一个浏览器或无痕窗口，用其他学员登录
3. 两个学员同时选择同一个排班，几乎同时点击"立即预约"
4. 观察：只有一个成功，另一个提示"该时间段已满员"

### 方法 C：使用脚本（推荐）

后端项目下创建测试脚本或直接使用 curl 批量：

```bash
# 获取两个 token
TOKEN1=$(curl -s -X POST http://localhost:28081/api/auth/login -H "Content-Type: application/json" -d '{"username":"student001","password":"123456"}' | grep -o '"token":"[^"]*"' | cut -d'"' -f4)

# 如果需要第二个学员账号，先通过管理员创建

# 同时发起预约（& 后台并发）
SCHEDULE_ID=<排班ID>
curl -s -X POST http://localhost:28081/api/reservations -H "Authorization: Bearer $TOKEN1" -H "Content-Type: application/json" -d "{\"scheduleId\":$SCHEDULE_ID}" &
# 如果有第二个 token，再发一个
# curl -s -X POST http://localhost:28081/api/reservations -H "Authorization: Bearer $TOKEN2" -H "Content-Type: application/json" -d "{\"scheduleId\":$SCHEDULE_ID}" &
wait
```

## 预期结果

1. **只有 1 个请求成功**，返回 `{"code":200,"message":"预约成功"}`
2. **其他请求失败**，返回：
   - `"该时间段已满员或不可预约"` 或
   - `"你已预约该时间段，不能重复预约"`
3. **数据库验证**：
   - `coach_schedule.current_students = 1`（不超过 max_students=1）
   - `reservation` 表中 SUCCESS 记录数 ≤ max_students

## 验证 SQL

```sql
-- 验证排班容量未超额
SELECT id, current_students, max_students
FROM coach_schedule WHERE id = <排班ID>;

-- 验证预约记录
SELECT COUNT(*) FROM reservation
WHERE schedule_id = <排班ID> AND status = 'SUCCESS';
```
