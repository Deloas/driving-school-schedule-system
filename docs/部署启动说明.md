# 部署启动说明

## 环境要求

| 软件 | 版本 | 说明 |
|------|------|------|
| JDK | 17 或 21 | java -version |
| Node.js | 20+ | node -v |
| Docker Desktop | 最新版 | docker --version |
| Maven | 3.9+ | mvn -version |
| IDE | IntelliJ IDEA | 推荐 |

## 端口分配

| 服务 | 端口 | 说明 |
|------|------|------|
| 后端 API | 28081 | Spring Boot |
| 前端 | 25173 | Vite Dev Server |
| MySQL | 23306 | Docker 映射 |
| Redis | 26379 | Docker 映射 |

## 启动顺序

### 1. 启动 Docker Desktop

确保 Docker 图标显示运行状态。

### 2. 启动 MySQL + Redis

```bash
cd D:/DrivingStSystem
docker-compose up -d
```

验证：

```bash
docker ps
# 应看到 driving-mysql 和 driving-redis 两个容器
```

### 3. 初始化数据库

首次启动时 `sql/init.sql` 会自动挂载并执行。
如果未自动执行，手动：

```bash
docker exec -i driving-mysql mysql -u root -p123456 < sql/init.sql
```

测试数据由 `DataInitializer`（仅 dev 环境）在应用启动时自动创建，包括：
- 5 名教练
- 6 辆车辆
- 20 名学员
- 系统账号（admin, coach001-005, student001-003）
- 排班测试数据

### 4. IDEA 启动后端

1. File → Open → 选择项目根目录（或 backend/）
2. 等待 Maven 导入依赖
3. Project Structure → SDK → JDK 17/21
4. 找到 `DrivingSchoolApplication.java`
5. 右键 → Run

日志中看到 `Started DrivingSchoolApplication` 表示成功。

验证：http://localhost:28081/api/health

### 5. 启动前端

```bash
cd frontend
npm install    # 首次执行
npm run dev
```

浏览器访问：http://localhost:25173

## IDEA 启动注意事项

1. 确保 JDK 配置正确（File → Project Structure → SDK）
2. 确保 `application-dev.yml` 中 MySQL/Redis 端口与 docker-compose 一致
3. 如果修改了代码，Stop → Run 完全重启（不要热加载）
4. 如果出现 `BeanCreationException`，检查 MySQL/Redis 是否已启动
5. 如果出现 500 接口错误，先检查 IDEA Console 堆栈信息

## 常见启动问题

| 问题 | 原因 | 解决 |
|------|------|------|
| Docker 启动失败 | Docker Desktop 未运行 | 启动 Docker Desktop |
| MySQL 连接失败 | 端口被占用 | `netstat -ano|findstr 23306` 查看，改为其他端口 |
| Redis 连接失败 | 同上 | 同法处理 |
| 后端 500 | 未加载最新代码 | IDEA Stop → Run |
| 前端白屏 | npm 依赖未安装 | `npm install` |
| favicon.ico 404 | 正常，可忽略 | - |
| Using generated password | 已被 SecurityConfig 替代，正常 | - |
