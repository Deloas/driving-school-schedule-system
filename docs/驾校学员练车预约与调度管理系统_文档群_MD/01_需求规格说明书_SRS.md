# 01 需求规格说明书 SRS

## 1. 引言

### 1.1 编写目的

本文档用于明确驾校学员练车预约与调度管理系统的功能需求、非功能需求、业务边界和验收标准，为后续系统设计、编码实现、测试验收和答辩展示提供依据。

### 1.2 项目定位

本系统是一个面向驾校内部练车培训过程的信息化系统，重点解决学员练车预约混乱、教练工作量不均衡、车辆资源利用不合理等问题。

系统不是考试管理系统，不负责考试预约、考试成绩、交管数据对接等内容。

## 2. 业务角色需求

### 2.1 学员需求

| 编号 | 需求 |
|---|---|
| S-01 | 学员可以登录系统查看个人信息 |
| S-02 | 学员可以查看自己的主教练 |
| S-03 | 学员可以查看某日期可预约时间段 |
| S-04 | 学员可以优先预约自己的主教练 |
| S-05 | 当主教练满员时，学员可以选择调剂到其他教练 |
| S-06 | 学员可以取消未开始的预约 |
| S-07 | 学员可以查看历史预约记录 |
| S-08 | 学员可以查看已完成练车次数 |

### 2.2 教练需求

| 编号 | 需求 |
|---|---|
| C-01 | 教练可以登录系统查看自己的排班 |
| C-02 | 教练可以查看某日某时间段预约学员名单 |
| C-03 | 教练可以查看由其他教练调剂过来的学员 |
| C-04 | 教练可以确认学员是否到场 |
| C-05 | 教练可以确认本次练车是否完成 |
| C-06 | 教练可以填写训练内容和备注 |

### 2.3 管理员需求

| 编号 | 需求 |
|---|---|
| A-01 | 管理员可以维护学员信息 |
| A-02 | 管理员可以维护教练信息 |
| A-03 | 管理员可以维护车辆信息 |
| A-04 | 管理员可以设置教练排班 |
| A-05 | 管理员可以设置每个教练每个时间段最大预约人数 |
| A-06 | 管理员可以查看全部预约记录 |

## 3. 功能需求

### 3.1 登录认证

- 支持账号密码登录；
- 支持角色区分；
- 登录成功返回 Token；
- 前端根据角色进入不同首页；
- 后端接口必须进行权限校验。

### 3.2 学员管理

- 新增学员；
- 编辑学员；
- 删除或禁用学员；
- 查询学员；
- 为学员分配主教练；
- 查看学员练车次数；
- 查看学员预约历史。

### 3.3 教练管理

- 新增教练；
- 编辑教练；
- 禁用教练；
- 设置默认半天最大带练人数；
- 查看教练负责学员；
- 查看教练工作量。

### 3.4 车辆管理

- 新增车辆；
- 编辑车辆；
- 设置车辆状态；
- 绑定默认教练；
- 查看车辆使用记录。

车辆状态建议：

| 状态 | 含义 |
|---|---|
| NORMAL | 正常 |
| MAINTENANCE | 维修中 |
| STOPPED | 停用 |

### 3.5 排班管理

排班是预约的前置条件。

管理员需要为教练设置：

- 日期；
- 时间段；
- 最大预约人数；
- 可预约状态；
- 绑定可用车辆；
- 备注。

时间段建议：

| 值 | 说明 |
|---|---|
| MORNING | 上午 |
| AFTERNOON | 下午 |

全天预约可理解为同时占用上午和下午两个时间段。

### 3.6 预约管理

预约规则：

1. 学员只能预约未来时间段；
2. 学员同一时间段只能有一个有效预约；
3. 教练该时间段必须有排班；
4. 教练当前预约人数不能超过最大容量；
5. 车辆必须可用；
6. 主教练有名额时优先预约主教练；
7. 主教练满员时，可以调剂到其他空闲教练；
8. 取消预约后释放容量；
9. 完成练车后生成练车记录。

### 3.7 调剂管理

调剂场景：

- 学员的主教练某时间段已满；
- 学员该时间段确实想练车；
- 其他教练有空余名额；
- 系统推荐空闲教练；
- 学员确认后生成调剂预约。

调剂记录应保存：

- 学员 ID；
- 主教练 ID；
- 实际教练 ID；
- 是否调剂；
- 调剂原因；
- 创建时间。

### 3.8 练车记录

教练在练车结束后可以确认：

- 学员是否到场；
- 是否完成本次训练；
- 训练内容；
- 教练备注；
- 使用车辆；
- 练车时间段。

完成后：

- 预约状态改为 COMPLETED；
- 学员 completed_training_count 加 1；
- 生成 training_record 记录。

## 4. 非功能需求

| 类别 | 要求 |
|---|---|
| 性能 | 预约查询接口响应尽量小于 500ms |
| 并发 | 同一排班名额并发预约时不能超额 |
| 安全 | 接口必须鉴权，敏感接口按角色授权 |
| 可维护 | 代码有清晰分层，关键代码中文注释 |
| 可扩展 | 后续可扩展候补队列、通知、学时统计 |
| 可演示 | 至少支持管理员、教练、学员三类账号演示 |
| 易用性 | 前端界面应清晰展示剩余名额与调剂选择 |

## 5. 业务边界

本系统只负责驾校内部练车过程管理，不处理正式考试安排，不替代交管考试系统，不提供考试成绩查询。
