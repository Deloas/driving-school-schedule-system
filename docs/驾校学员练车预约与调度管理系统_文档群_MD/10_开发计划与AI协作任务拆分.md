# 10 开发计划与 AI 协作任务拆分

## 1. 开发原则

1. 先跑通主链路，再做界面美化；
2. 先做后端核心业务，再做统计；
3. 前端必须围绕预约体验重点设计；
4. AI 每次只完成一个明确任务；
5. 每完成一个任务都要运行项目并修复错误；
6. 代码必须有中文注释；
7. 不做考试功能，不做偏题功能。

## 2. 里程碑规划

### M1：项目初始化

目标：

- 创建前后端项目；
- 配置数据库；
- 配置基础目录结构；
- 配置统一响应和异常处理。

产出：

- Spring Boot 项目；
- Vue 3 项目；
- MySQL 建表脚本；
- 基础 README。

### M2：用户认证与权限

目标：

- 实现登录；
- 实现 JWT；
- 实现角色权限；
- 前端角色路由。

产出：

- 登录接口；
- 登录页；
- Token 拦截器；
- 权限路由。

### M3：基础资料管理

目标：

- 学员管理；
- 教练管理；
- 车辆管理。

产出：

- 后端 CRUD；
- 前端列表页和表单；
- 数据校验。

### M4：排班管理

目标：

- 管理员创建教练排班；
- 支持上午/下午；
- 支持容量设置；
- 支持车辆绑定。

产出：

- 排班表；
- 排班接口；
- 排班管理页。

### M5：预约主流程

目标：

- 学员查询可预约时间；
- 学员预约主教练；
- 防重复预约；
- 防容量超额。

产出：

- 预约接口；
- 学员预约页；
- 预约记录页。

### M6：调剂机制

目标：

- 主教练满员后推荐其他教练；
- 学员确认调剂；
- 保存 main_coach_id 和 actual_coach_id；
- 统计调剂数量。

产出：

- 调剂推荐接口；
- 调剂预约接口；
- 调剂推荐 UI。

### M7：练车完成与统计

目标：

- 教练查看名单；
- 教练确认到场；
- 教练完成练车；
- 学员练车次数累计；
- 管理员统计看板。

产出：

- 教练工作台；
- 练车记录；
- 数据看板。

### M8：产品级优化与答辩准备

目标：

- 前端高级感优化；
- 演示数据准备；
- 修复 Bug；
- 编写答辩脚本。

产出：

- 演示账号；
- 演示流程；
- 答辩 PPT 素材。

## 3. AI 编程助手任务拆分

### 任务 1：初始化后端项目

给 AI 的任务：

```text
请创建 Spring Boot 3.x 后端项目，使用 Java 17，包名 com.drivingschool。配置 MyBatis-Plus、MySQL、Redis、Spring Security、JWT。请生成标准分层目录，并添加统一响应 Result、全局异常处理、基础枚举。所有类和关键方法添加中文注释。
```

### 任务 2：生成数据库脚本和实体

```text
请根据 docs/05_数据库设计.md 生成 MySQL 建表脚本、Java Entity、Mapper、基础枚举。字段必须对应数据库表，实体类字段添加中文注释。
```

### 任务 3：实现登录权限

```text
请实现登录接口、JWT 生成与解析、Spring Security 鉴权、角色权限控制。管理员、教练、学员登录后返回不同角色信息。关键安全逻辑添加中文注释。
```

### 任务 4：实现基础资料 CRUD

```text
请实现学员、教练、车辆的 CRUD 接口和分页查询。注意学员必须绑定主教练，车辆可绑定默认教练。所有业务校验返回中文错误提示。
```

### 任务 5：实现排班管理

```text
请实现教练排班模块，包括创建排班、批量生成排班、查询可预约排班、关闭排班。必须保证同一教练同一天同一时间段不能重复排班。
```

### 任务 6：实现预约核心逻辑

```text
请实现学员预约练车接口。规则：优先主教练；不能重复预约；不能超过排班容量；排班必须开放；车辆必须正常；预约成功后排班当前人数加 1。请使用事务，并为核心业务代码添加详细中文注释。
```

### 任务 7：实现 Redis 分布式锁防超额

```text
请为预约接口加入 Redisson 分布式锁，锁粒度为 scheduleId。锁内重新查询数据库，并使用数据库条件更新 current_students < max_students 作为兜底，确保并发下不会超额预约。
```

### 任务 8：实现调剂机制

```text
请实现主教练满员后的调剂推荐接口。推荐同日期同时间段仍有剩余名额的其他教练，按剩余名额和工作量排序。创建调剂预约时保存 main_coach_id、actual_coach_id、is_adjusted。
```

### 任务 9：实现教练工作台与练车记录

```text
请实现教练查看当天预约名单、确认到场、完成练车接口。完成练车后生成 training_record，并让学员 completed_training_count 加 1。
```

### 任务 10：实现前端产品级界面

```text
请使用 Vue 3 + Vite + TypeScript + Tailwind CSS 实现前端。必须采用 Anthropic frontend-design 的设计思路，设计成高级、专业、具有调度系统感的界面。不要使用普通后台模板。实现登录页、管理员首页、学员预约页、教练工作台、排班管理页、统计页。关键组件添加中文注释。
```

## 4. 每次让 AI 开发后的检查清单

每完成一个任务，要求 AI：

1. 说明修改了哪些文件；
2. 说明实现了哪些接口或页面；
3. 给出运行命令；
4. 给出测试方法；
5. 检查是否有中文注释；
6. 检查是否偏离“不做考试”的系统边界；
7. 修复编译错误和类型错误。
