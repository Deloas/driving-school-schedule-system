# 03 产品原型与前端设计规范

## 1. 前端设计目标

本项目不是普通后台管理模板，而应体现“产品级、高级感、真实可用”的系统质感。

设计关键词：

- 清晰；
- 专业；
- 高级；
- 信息密度合理；
- 驾校业务感；
- 调度系统感；
- 避免千篇一律的白底蓝色后台模板。

## 2. Anthropic frontend-design 使用要求

前端开发必须使用 Anthropic frontend-design 的设计思想：

1. 先确定视觉性格，再写代码；
2. 避免普通 AI 生成界面的模板感；
3. 在字体、颜色、动效、卡片、背景、数据展示上有明确设计意图；
4. 输出可运行、可维护、产品级代码；
5. 组件要像真实 SaaS 产品，而不是课程作业页面。

如果 AI 编程工具支持 Claude Skill，请启用：

```text
frontend-design
```

如果不支持 Skill，则在提示词中明确要求：

```text
请采用 Anthropic frontend-design 的思路设计前端：
- 设计要有明确视觉方向；
- 避免普通后台模板；
- 使用克制但高级的视觉系统；
- 对排班、预约、容量、调剂等业务状态进行可视化表达；
- 所有前端关键逻辑添加中文注释。
```

## 3. 推荐视觉方向

### 3.1 视觉概念

建议采用：

> 智能调度驾驶舱 + 清爽业务管理台

整体不要做成游戏风，也不要做成花哨官网。应该像一个真实驾校内部使用的轻量 SaaS 系统。

### 3.2 色彩建议

| 用途 | 建议 |
|---|---|
| 主色 | 深青绿、墨蓝、驾驶舱蓝、道路绿 |
| 辅助色 | 温和橙色用于提醒，绿色用于可预约，红色用于满员 |
| 背景 | 浅灰白或低饱和冷色背景 |
| 卡片 | 半透明或柔和阴影，边界清晰 |
| 状态 | OPEN、FULL、CANCELLED、COMPLETED 需要颜色区分 |

### 3.3 页面气质

- 顶部导航简洁；
- 侧边栏图标清晰；
- 数据卡片有层次；
- 预约容量用进度条表达；
- 排班用日历/时间段卡片表达；
- 调剂推荐用对比卡片表达；
- 数据看板使用折线图、柱状图、环形图。

## 4. 前端技术栈

推荐：

- Vue 3
- Vite
- TypeScript
- Tailwind CSS
- Pinia
- Vue Router
- Axios
- ECharts 或 Recharts 类图表库
- Element Plus 或 Naive UI 可选

说明：

如果使用 Element Plus，必须进行二次设计，不要直接套默认样式。

## 5. 页面清单

### 5.1 登录页

功能：

- 输入账号；
- 输入密码；
- 选择或自动识别角色；
- 登录按钮；
- 展示系统名称和一句价值主张。

价值主张示例：

> 让每一次练车预约都有秩序，让每一位教练负载更均衡。

### 5.2 管理员首页

模块：

- 今日预约人数；
- 今日可用教练数；
- 今日可用车辆数；
- 已满排班数；
- 教练负载排行；
- 预约趋势图；
- 调剂预约数量。

### 5.3 学员预约页

核心页面，必须重点设计。

内容：

- 日期选择；
- 上午/下午/全天选择；
- 我的主教练卡片；
- 主教练剩余名额；
- 可调剂教练列表；
- 预约按钮；
- 预约规则提示。

视觉建议：

- 主教练用主卡片展示；
- 剩余名额用容量进度条；
- 满员时显示“推荐调剂方案”；
- 调剂教练卡片展示剩余名额、今日负载、车辆状态。

### 5.4 教练工作台

内容：

- 今日排班；
- 今日预约学员；
- 调剂学员标识；
- 到场确认；
- 完成练车；
- 填写训练内容；
- 工作量统计。

### 5.5 排班管理页

内容：

- 日期筛选；
- 教练筛选；
- 批量生成排班；
- 设置上午/下午容量；
- 绑定车辆；
- 开启/关闭预约。

### 5.6 车辆管理页

内容：

- 车辆列表；
- 车牌号；
- 绑定教练；
- 车辆状态；
- 维修标记；
- 使用记录。

### 5.7 数据统计页

内容：

- 每日预约人数趋势；
- 教练工作量对比；
- 调剂次数统计；
- 学员练车次数分布；
- 车辆利用率。

## 6. 交互细节

### 6.1 预约按钮状态

| 状态 | 表现 |
|---|---|
| 可预约 | 按钮高亮 |
| 已满员 | 按钮禁用，显示可调剂 |
| 未排班 | 显示“该时间段暂无排班” |
| 已预约 | 显示“你已预约该时间段” |

### 6.2 调剂提示文案

```text
你的主教练当前时间段已满员，系统为你找到 3 位仍有空余名额的教练。调剂练车不会改变你的主教练归属。
```

### 6.3 取消预约提示

```text
取消后该名额将释放给其他学员，请确认是否取消本次练车预约？
```

## 7. 组件设计建议

| 组件 | 说明 |
|---|---|
| CapacityBar | 展示已预约/最大容量 |
| CoachCard | 展示教练、剩余名额、负载 |
| ScheduleSlot | 展示上午/下午排班块 |
| ReservationTimeline | 展示预约进度 |
| AdjustmentPanel | 展示调剂推荐 |
| StatCard | 数据看板卡片 |
| StatusBadge | 状态标签 |
| EmptyState | 空数据提示 |

## 8. 中文注释要求

前端代码中必须包含中文注释：

- 复杂组件顶部说明用途；
- 关键状态字段说明；
- 预约提交前校验说明；
- 调剂推荐展示逻辑说明；
- API 封装说明；
- 权限路由守卫说明。

示例：

```ts
// 判断当前时间段是否还有剩余名额，避免学员点击已满员的排班
const hasRemain = computed(() => props.slot.currentCount < props.slot.maxCount)
```
