## 兜底方案

### 前端登录失败
1. 确认后端已启动: curl http://localhost:28081/api/health
2. 检查账号是否被停用: admin 登录 → 账号管理 → 启用
3. 重置密码: 账号管理 → 重置密码

### Redis 未启动
1. docker ps | grep driving-redis
2. 如果没有: docker-compose up -d redis
3. 重启后端

### 并发脚本失败
1. 确认 PowerShell 版本: $PSVersionTable
2. 确认后端可访问: curl http://localhost:28081/api/health
3. 如果 ScheduleId 未找到，手动查询排班 ID 并修改脚本中的 $ScheduleId

### 数据不干净
1. 执行 05_reset_demo_scene.sql 重置
2. 重新执行 00 → 01 → 02

### 页面无数据
1. 检查筛选条件是否过滤了数据
2. 检查日期是否在排班数据范围内
3. 重新执行 DataInitializer（重启后端即可）

### 端口被占用
```bash
# 查看占用端口的进程
netstat -ano | findstr "28081"
netstat -ano | findstr "25173"
netstat -ano | findstr "23306"
netstat -ano | findstr "26379"
```

### student002 密码不对
1. admin 登录
2. 账号管理 → 找到 student002 → 重置密码

### 现场网络不可用
- 所有演示基于本地 localhost，不依赖外网
- 提前准备离线环境
- IDEA 和 Docker Desktop 提前打开
