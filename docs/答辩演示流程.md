# 答辩演示流程

## 演示前准备

```bash
# 1. 启动 Docker Desktop
# 2. 启动 MySQL + Redis
docker-compose up -d

# 3. IDEA 完全 Stop → Run 启动 DrivingSchoolApplication

# 4. 启动前端
cd frontend && npm run dev
```

- 后端: http://localhost:28081
- 前端: http://localhost:25173

## 演示步骤（8-10 分钟）

### 第一步：管理员登录与驾驶舱 (1 分钟)

1. 打开 http://localhost:25173
2. 点击「管理员」→ admin / 123456 → 登录
3. 进入管理员驾驶舱首页

**展示点：**
- 登录页产品级设计
- 统计看板：学员/教练/车辆总数、今日预约、教练负载
- 说明系统可掌握驾校全局运行情况

### 第二步：管理员查看基础资料 (1 分钟)

1. 侧边栏 → 教练管理：展示 5 名教练
2. 学员管理：展示 20 名学员，每名学员绑定主教练
3. 车辆管理：展示 6 辆车，绑定教练

**展示点：**
- 基础资料维护页面产品级设计
- 学员绑定主教练关系
- 车辆状态管理

### 第三步：查看排班 (0.5 分钟)

1. 侧边栏 → 排班管理
2. 展示排班列表，可选日期筛选
3. 点击「批量生成」展示批量排班能力

**展示点：**
- 排班按日期+上午/下午+教练+车辆组织
- 容量控制：max_students 限制每时段人数
- 批量生成：多教练×多天×多时段

### 第四步：学员预约练车 (1.5 分钟)

1. 退出管理员，student001 登录
2. 进入「预约练车」页面
3. 选择日期和上午时段 → 查询
4. 页面显示主教练唐国义，剩余名额 5 → 可预约
5. 点击「立即预约」→ 预约成功
6. 页面显示「我的预约记录」

**展示点：**
- 学员工作台 Hero + 统计卡片
- 主教练卡片：姓名+车辆+容量+进度条
- 预约成功即时反馈
- 我的预约列表刷新

### 第五步：演示满员调剂 (1.5 分钟)

1. 提前将唐国义某天上午排班设为 max=1, current=1
2. 确保张三同天上午有 OPEN 排班有剩余名额
3. student001 选择该日期上午 → 查询
4. 显示主教练已满员 → 下方出现「可调剂教练推荐」
5. 张三卡片显示剩余名额
6. 点击「选择调剂」→ 确认提示 → 调剂预约成功
7. 我的预约列表显示「调剂」标记 + 实际教练张三 + 主教练唐国义

**展示点：**
- 调剂推荐按剩余名额排序
- 调剂不改变主教练归属（main_coach_id 不变）
- 预约记录中区分主教练和实际教练

### 第六步：教练完成练车 (1.5 分钟)

1. 退出学员，coach002 登录
2. 进入教练工作台
3. 显示「今日预约名单」→ 包含 student002 的预约
4. 点击某条待练车预约的「完成」按钮
5. 弹窗填写：练车内容「倒车入库练习」+ 时长 120 分钟 + 备注「倒库需加强」
6. 确认完成 → 预约状态变为「已完成」
7. 教练工作台统计数字更新

**展示点：**
- 教练工作台 Hero + 统计卡片 + 日期筛选
- 「我的学员」区块
- 完成练车弹窗
- 状态实时更新

### 第七步：学员查看练车档案 (0.5 分钟)

1. 退出教练，student002 登录
2. 点击「练车记录」
3. 显示「我的训练档案」
4. 每条记录展示：日期+时段+教练+车辆+内容+教练评语

**展示点：**
- 训练档案卡片式设计
- 练车内容和教练评语沉淀
- 空状态友好提示

### 第八步：管理员总结 (0.5 分钟)

1. admin 登录 → 驾驶舱首页
2. 查看统计：今日预约、今日完成、教练负载等数据变化
3. 查看练车记录 → 全部记录列表

**展示点：**
- 数据随预约完成实时变化
- 完整的预约→练车→记录闭环

## 常见问题与兜底

| 问题 | 兜底方案 |
|------|---------|
| Redis/MySQL 未启动 | 执行 `docker-compose up -d` |
| 后端 500 | IDEA Stop → Run 完全重启 |
| 前端白屏 | `npm run dev` 重新启动 |
| 测试数据不足 | 后端启动后 DataInitializer 自动创建 |
| 密码错误 | admin 登录 → 账号管理 → 重置密码 |
