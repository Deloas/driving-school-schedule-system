# 数据库设计说明

## 数据库信息

- 数据库名: `driving_school_schedule`
- 字符集: `utf8mb4`
- 共 8 张核心表

## 核心表清单

### 1. sys_user — 系统用户表

存储登录账号、BCrypt 密码、角色、绑定关系。

| 字段 | 说明 |
|------|------|
| id | 主键 |
| username | 登录账号，唯一 |
| password | BCrypt 加密密码 |
| role | ADMIN / COACH / STUDENT |
| related_id | 绑定 coach.id 或 student.id |
| status | ENABLE / DISABLE |

### 2. coach — 教练表

教练基本信息和默认带教容量。

| 字段 | 说明 |
|------|------|
| id | 主键 |
| name | 教练姓名 |
| phone | 手机号 |
| license_no | 教练证号 |
| max_students_per_half_day | 默认每半天最大带练人数 |
| status | NORMAL / LEAVE / STOPPED |

### 3. student — 学员表

学员信息和练车进度。

| 字段 | 说明 |
|------|------|
| id | 主键 |
| name | 学员姓名 |
| phone | 手机号 |
| main_coach_id | 主教练 ID（关键外键） |
| subject_type | SUBJECT_2 / SUBJECT_3 |
| required_training_count | 应完成练车次数 |
| completed_training_count | 已完成练车次数 |
| status | NORMAL / STOPPED |

### 4. vehicle — 车辆表

训练车辆信息和状态。

| 字段 | 说明 |
|------|------|
| id | 主键 |
| plate_number | 车牌号，唯一 |
| coach_id | 默认绑定教练 |
| vehicle_type | C1 / C2 |
| status | NORMAL / MAINTENANCE / STOPPED |

### 5. coach_schedule — 教练排班表

某教练在某日期某时间段的排班，含容量控制。

| 字段 | 说明 |
|------|------|
| id | 主键 |
| coach_id | 教练 ID |
| vehicle_id | 绑定车辆 |
| schedule_date | 排班日期 |
| time_slot | MORNING / AFTERNOON |
| max_students | 最大预约人数 |
| current_students | 当前预约人数（并发控制关键） |
| status | OPEN / CLOSED / CANCELLED |

> 唯一约束: (coach_id, schedule_date, time_slot)

### 6. reservation — 预约表

学员预约记录，核心表。

| 字段 | 说明 |
|------|------|
| id | 主键 |
| student_id | 学员 ID |
| main_coach_id | 学员主教练 ID（归属不变） |
| actual_coach_id | 本次实际带练教练 ID（可能不同） |
| schedule_id | 关联排班 ID |
| reservation_date | 预约日期 |
| time_slot | MORNING / AFTERNOON |
| status | SUCCESS / CANCELLED / COMPLETED / ABSENT |
| is_adjusted | 是否调剂（0/1） |
| adjust_reason | 调剂原因 |
| cancel_reason | 取消原因 |
| cancel_time | 取消时间 |

> 关键：main_coach_id 和 actual_coach_id 分离，体现主教练负责制和调剂机制。

### 7. training_record — 练车记录表

预约完成后的实际训练记录。

| 字段 | 说明 |
|------|------|
| id | 主键 |
| reservation_id | 关联预约 ID |
| student_id | 学员 ID |
| coach_id | 实际带练教练 |
| vehicle_id | 训练车辆 |
| training_date | 训练日期 |
| time_slot | 时间段 |
| training_content | 训练内容 |
| coach_comment | 教练备注 |
| result | COMPLETED / ABSENT |

### 8. operation_log — 操作日志表

审计操作记录。

| 字段 | 说明 |
|------|------|
| id | 主键 |
| user_id | 操作用户 |
| operation_type | 操作类型 |
| business_type | 业务类型 |
| content | 操作内容 |
| create_time | 操作时间 |

## 表关系

```
sys_user.related_id → coach.id (role=COACH)
sys_user.related_id → student.id (role=STUDENT)

student.main_coach_id → coach.id

vehicle.coach_id → coach.id

coach_schedule.coach_id → coach.id
coach_schedule.vehicle_id → vehicle.id

reservation.student_id → student.id
reservation.main_coach_id → coach.id (主教练)
reservation.actual_coach_id → coach.id (实际教练)
reservation.schedule_id → coach_schedule.id
reservation.vehicle_id → vehicle.id

training_record.reservation_id → reservation.id
training_record.student_id → student.id
training_record.coach_id → coach.id
training_record.vehicle_id → vehicle.id
```

## 关键设计：主教练与调剂教练

- `reservation.main_coach_id` = 学员归属的主教练（不变）
- `reservation.actual_coach_id` = 本次实际带练的教练
- `reservation.is_adjusted` = 1 表示调剂预约
- 调剂后 `main_coach_id != actual_coach_id`
