# 答辩小组分工说明

## 分工原则

按**业务功能模块**划分，不按"产品/后端/前端/测试"横向拆层。每位成员对自己的功能模块从数据库到前端都能讲解。

## 五人分工表

| 成员 | 负责模块 | 核心内容 |
|------|---------|---------|
| A | 登录认证与账号权限 | JWT、Spring Security、角色权限、账号管理、401/403 |
| B | 基础资料管理 | 教练/学员/车辆 CRUD、主教练绑定、车辆绑定 |
| C | 排班管理 | 排班创建/编辑/批量生成、日期时段、容量设置 |
| D | 学员预约与调剂 | 预约/取消、Redis 锁、条件UPDATE、满员调剂 |
| E | 练车闭环与统计看板 | 完成/缺席、training_record、训练档案、统计驾驶舱 |

## 成员 A：登录认证与账号权限模块

**负责模块：**
- 登录认证（POST /api/auth/login、GET /api/auth/me）
- JWT Token 生成与校验
- Spring Security 配置 + JwtAuthenticationFilter
- 角色权限（ADMIN / COACH / STUDENT）
- 账号管理（GET/POST /api/users）
- 账号与人员绑定（sys_user.related_id → coach.id / student.id）
- 停用账号拦截登录
- 401 / 403 统一 JSON 返回

**负责页面：**
- 登录页
- 账号管理页（/admin/users）

**涉及数据库表：**
- sys_user

**答辩讲解重点：**
- 登录流程：BCrypt 校验 → 生成 JWT → 返回 token/role/relatedId
- 权限设计原则：功能模块包含权限分级
- @PreAuthorize + ROLE_ 前缀机制
- 账号与教练/学员一对一绑定
- AccessDeniedException → 403 JSON

## 成员 B：基础资料管理模块

**负责模块：**
- 教练管理（CRUD + 状态 + 带教容量）
- 学员管理（CRUD + 主教练绑定 + 练车进度）
- 车辆管理（CRUD + 教练绑定 + 状态）
- 教练/学员简单列表接口（下拉选择用）

**负责页面：**
- 教练管理页（/admin/coaches）
- 学员管理页（/admin/students）
- 车辆管理页（/admin/vehicles）

**涉及数据库表：**
- coach
- student
- vehicle

**答辩讲解重点：**
- 学员与主教练的绑定关系（main_coach_id）
- 教练带教容量（max_students_per_half_day）
- 车辆绑定教练 + 维修/停用状态管理
- 教练查看自己名下学员（GET /api/students/my）
- 状态筛选、分页、编辑功能

## 成员 C：排班管理模块

**负责模块：**
- 排班创建（POST /api/schedules）
- 排班编辑（PUT /api/schedules/{id}）
- 排班状态管理（PATCH /api/schedules/{id}/status）
- 批量生成排班（POST /api/schedules/batch）
- 可预约排班查询（GET /api/schedules/available）
- 排班容量设置（max_students）
- 教练重复排班校验
- 车辆冲突校验

**负责页面：**
- 排班管理页（/admin/schedules）

**涉及数据库表：**
- coach_schedule

**答辩讲解重点：**
- 排班 = 日期 + 时段(上午/下午) + 教练 + 车辆 + 容量
- 批量生成：多教练 × 多天 × 多时段，自动跳过重复
- 唯一约束：同一教练同一天同一时间段只有一个排班
- OPEN/CLOSED/CANCELLED 状态流转
- 容量不能小于已预约人数（编辑时校验）

---

## 成员 D：学员预约与调剂模块

**负责模块：**
- 学员查询预约方案（GET /api/reservations/options）
- 创建预约（POST /api/reservations）
- 取消预约（POST /api/reservations/{id}/cancel）
- 重复预约拦截
- Redis 分布式锁（lock:schedule:{id}）
- 数据库条件 UPDATE 兜底
- 防重复点击限流（5秒）
- 主教练满员调剂推荐
- 调剂预约（adjusted=true, is_adjusted=1）
- main_coach_id 与 actual_coach_id 分离设计

**负责页面：**
- 学员预约练车页（/student/reservations）

**涉及数据库表：**
- reservation
- coach_schedule（current_students 更新）

**答辩讲解重点：**
- 预约流程：选日期→选时段→查主教练→容量校验→预约
- Redis 锁流程：tryLock→锁内重查→条件UPDATE→Insert→释放
- 并发安全演示：两个学员同时抢最后一个名额
- 调剂：主教练满员→推荐其他教练→调剂预约
- main_coach_id ≠ actual_coach_id 体现设计

---

## 成员 E：练车闭环与统计看板模块

**负责模块：**
- 教练完成练车（POST /api/reservations/{id}/complete）
- 标记缺席（POST /api/reservations/{id}/absent）
- 训练记录生成（training_record）
- 学员练车次数累加（completed_training_count）
- 学员训练档案（GET /api/training-records/my）
- 教练练车记录（GET /api/training-records/coach）
- 管理员练车记录（GET /api/training-records）
- 管理员统计驾驶舱（GET /api/statistics/admin/*）
- 教练个人看板（GET /api/statistics/coach/overview）
- 学员个人看板（GET /api/statistics/student/overview）

**负责页面：**
- 教练工作台（/coach）
- 学员训练档案（/student/training-records）
- 管理员统计驾驶舱（/admin）
- 管理员练车记录（/admin/training-records）

**涉及数据库表：**
- reservation
- training_record
- student（completed_training_count 更新）

**答辩讲解重点：**
- 完成练车事务：COMPLETED + training_record + 次数+1
- 缺席处理：ABSENT + 不生成记录 + 不累加次数
- 训练档案：卡片式展示训练内容和教练评语
- 统计驾驶舱：教练负载、预约趋势、容量利用率
- 练车闭环：预约→到场→完成→记录→统计
