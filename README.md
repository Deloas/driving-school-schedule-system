# 驾校学员练车预约与调度管理系统

> 基于教练资源均衡的智能练车预约平台

## 项目简介

本系统围绕驾校内部学员练车管理，解决传统电话预约和微信群接龙带来的预约混乱、教练负载不均等问题。

**核心问题解决：**

| 传统问题 | 系统方案 |
|---------|---------|
| 电话预约/微信群混乱 | 在线按日期和时段预约 |
| 教练工作量不均 | 排班容量控制 + 负载统计 |
| 主教练满员后无方案 | 调剂推荐 + 临时调剂预约 |
| 预约超额 | Redis 锁 + 数据库条件 UPDATE |
| 练车无记录 | 训练记录生成 + 训练档案 |

## 项目定位

本项目是**驾校内部练车预约与调度管理系统**，不做考试、不做成绩、不做交管。

| 系统做 | 系统不做 |
|--------|---------|
| 学员/教练/车辆管理 | 考试预约 |
| 教练排班 + 批量生成 | 考试成绩 |
| 学员在线预约练车 | 交管系统对接 |
| Redis 锁防超额 | 缴费 |
| 主教练满员调剂 | 合同管理 |
| 教练完成/缺席 | 驾考全流程 |
| 训练记录沉淀 | |
| 统计驾驶舱 | |

## 核心功能

| 模块 | 说明 |
|------|------|
| 登录认证 | JWT Token + 角色路由守卫 |
| 账号管理 | 为教练/学员开通登录账号 |
| 教练/学员/车辆 | 基础资料 CRUD |
| 排班管理 | 按日期+时段+教练+车辆创建排班，批量生成 |
| 学员预约 | 按日期和时段预约主教练练车 |
| Redis 并发锁 | Redisson 锁保证不会超额 |
| 满员调剂 | 主教练满员→推荐其他教练→临时调剂 |
| 完成练车 | 教练确认→填写内容→生成训练记录 |
| 练车记录 | 学员档案、教练历史、管理员查询 |
| 统计看板 | 驾驶舱 + 教练负载 + 预约趋势 |

## 快速启动

```bash
# 1. 启动 MySQL + Redis
docker-compose up -d

# 2. IDEA 打开 backend/，运行 DrivingSchoolApplication

# 3. 启动前端
cd frontend && npm install && npm run dev
```

## 演示流程

1. 管理员登录 → 统计驾驶舱 → 查看教练/学员/车辆
2. 排班管理 → 查看或批量生成排班
3. 学员登录 → 查询主教练 → 预约练车
4. 主教练满员 → 调剂推荐 → 调剂预约
5. 教练登录 → 今日预约 → 完成练车 / 标记缺席
6. 学员查看训练档案 → 管理员查看统计

## 项目亮点

- **聚焦练车不做考试** — 严格遵循需求边界
- **主教练负责制** — 归属不变，调剂不改变归属
- **满员调剂** — 自动推荐可用教练
- **Redis 防超额** — 双重保障预约安全
- **完整闭环** — 预约→到场→完成→记录
- **产品级前端** — 三端品牌统一，Semi/Arco 风格
- **权限隔离** — 功能模块包含权限分级

## 系统边界

本系统不涉及：考试预约、考试成绩、交管系统对接、缴费、合同管理。
